The Illustrated Transformer 요약

Jay Alammar의 The Illustrated Transformer를 바탕으로, 트랜스포머의 핵심 아이디어를 기술적으로 의미 있게, 그러나 정보 밀도 높게 다시 정리한 문서입니다.

한 줄로 요약하면, 트랜스포머는 단어를 순서대로 읽는 모델에서 문장 전체를 한꺼번에 보고 필요한 단어끼리 직접 연결하는 모델로의 전환이었습니다.

1. 왜 Transformer가 중요한가

Transformer의 혁신은 RNN 없이도 시퀀스를 다룰 수 있다는 데 있습니다. 핵심 메커니즘은 self-attention이며, 각 토큰이 문장 안의 다른 모든 토큰을 직접 참고하게 만듭니다.

이 구조는 번역 문제를 위해 제안됐지만, 이후 BERT, GPT, T5, ViT로 확장되며 현대 AI의 공통 기반이 됐습니다.

주요 출처: The Illustrated Transformer, Attention Is All You Need

2. 입력 표현: 임베딩 + 위치 인코딩

트랜스포머는 먼저 단어를 임베딩 벡터로 바꿉니다. 예를 들어 “나는 사과를 먹었다”에서 사과는 단순 글자가 아니라, 과일·음식·먹다 같은 의미적 근접성을 반영한 숫자 벡터가 됩니다.

문제는 self-attention만으로는 순서 정보가 약하다는 점입니다. 그래서 각 토큰에 위치 인코딩(positional encoding)을 더해 “이 토큰이 몇 번째에 왔는지”를 알려 줍니다.

예시

  • “강가의 은행”과 “돈을 맡기는 은행”은 같은 단어지만, 주변 단어와 위치가 달라 문맥이 달라집니다.
  • 임베딩은 단어의 의미를, 위치 인코딩은 단어의 순서를 보완합니다.

출처: Attention Is All You Need, The Illustrated Transformer

3. 인코더: 입력 문장을 문맥 반영 표현으로 바꾸는 층

인코더는 문장 전체를 보고 각 단어를 문맥이 반영된 표현으로 바꿉니다. 각 층은 보통 multi-head self-attention + feed-forward network로 이루어지고, 여기에 residual connection과 normalization이 더해집니다.

예시

“동물은 길을 건너지 않았다, 너무 피곤했기 때문이다”에서 피곤했기 때문이 가리키는 것이 무엇인지 알려면 문장 전체를 봐야 합니다. 인코더는 이런 장거리 문맥 관계를 반영한 내부 표현을 만듭니다.

즉 인코더 출력은 단순 원문 복사가 아니라, 관계와 문맥이 압축된 표현입니다.

출처: The Illustrated Transformer, Attention Is All You Need

4. 디코더: 이전 출력과 원문 표현을 함께 보고 다음 토큰 생성

디코더는 번역문이나 응답문을 왼쪽에서 오른쪽으로 한 토큰씩 생성합니다. 구조적으로는:

  • Masked self-attention — 미래 토큰을 미리 못 보게 막음
  • Encoder-decoder attention — 원문 중 어떤 부분을 참고할지 선택
  • Feed-forward network

예시

번역 중 이미 The animal까지 생성했다면, 다음에 didn't cross가 자연스러운지 판단할 때 지금까지 생성한 출력과 인코더가 만든 원문 표현을 함께 봅니다.

출처: Attention Is All You Need, The Illustrated Transformer

5. 셀프 어텐션: 필요한 단어를 직접 참조하는 메커니즘

self-attention은 각 단어가 문장 안 다른 단어들을 얼마나 참고할지 가중치로 계산합니다. 핵심 수식은 다음과 같습니다.

Attention(Q, K, V) = softmax(QKᵀ / √dₖ)V

여기서 Q(Query)는 내가 찾는 정보, K(Key)는 내가 어떤 정보인지, V(Value)는 실제 가져갈 내용입니다.

예시

“The animal didn't cross the street because it was too tired.”에서 it가 street가 아니라 animal을 가리킨다는 점을 모델이 파악해야 합니다. self-attention은 it가 animal, tired, cross에 더 높은 가중치를 두게 만들 수 있습니다.

핵심은 RNN처럼 정보를 중간 상태에 눌러 담는 대신, 필요한 단어를 바로 참조한다는 점입니다.

출처: The Illustrated Transformer, Attention Is All You Need

6. 멀티헤드 어텐션: 여러 관점으로 동시에 읽기

한 개의 attention만 쓰면 관계 해석이 한 방향으로 치우칠 수 있습니다. multi-head attention은 여러 개의 head가 서로 다른 관계를 병렬로 봅니다.

예시

  • 한 head는 주어-동사 관계에 집중
  • 다른 head는 대명사-선행사 관계에 집중
  • 또 다른 head는 장거리 의미 연결에 집중

즉 모델이 문장을 문법·의미·거리 같은 여러 관점에서 동시에 읽게 됩니다.

출처: Attention Is All You Need, The Illustrated Transformer

7. 왜 RNN보다 강했나

  • 병렬화 가능: 문장 전체 토큰을 동시에 처리 가능
  • 장거리 의존성 경로 길이 1: 멀리 떨어진 단어도 바로 연결 가능
  • 대규모 학습과 GPU 친화성: 현대 하드웨어에 잘 맞음

예시

RNN은 첫 단어를 처리한 뒤에야 두 번째 단어로 갈 수 있지만, Transformer는 인코더 기준으로 문장 전체를 동시에 처리합니다. 그래서 대규모 번역 학습이나 언어모델 학습에 훨씬 유리했습니다.

출처: Attention Is All You Need

8. 복잡도 트레이드오프

Transformer는 병렬화에 강하지만, 기본 self-attention은 길이 n인 시퀀스에서 모든 토큰 쌍을 비교하므로 시간·메모리 비용이 대체로 O(n²)로 커집니다.

의미

  • 짧거나 중간 길이 문장, 대규모 병렬 학습에서는 매우 강력
  • 아주 긴 문맥에서는 attention 비용이 커져 후속 연구가 필요

즉 Transformer는 순차 병목을 없애는 대신, 토큰 간 비교 비용을 감수한 설계라고 볼 수 있습니다.

출처: Attention Is All You Need, PyTorch Tutorials, Hugging Face

9. 역사적 의미: BERT, GPT, T5, ViT

BERT — 인코더 계열의 대표

BERT는 트랜스포머의 인코더 구조를 중심으로 발전한 대표 사례입니다. 양방향 문맥 사전학습을 통해 문장 분류, 질의응답, 개체명 인식의 기준을 크게 끌어올렸습니다.

출처: BERT (ACL Anthology), BERT (arXiv)

GPT — 디코더 계열의 대표

GPT 계열은 트랜스포머의 디코더 구조를 대규모 언어모델로 확장한 흐름입니다. 다음 토큰 예측만으로 생성, 요약, 코드 작성, few-shot 학습이 가능해졌습니다.

출처: GPT-2, GPT-3

T5 — 모든 NLP를 텍스트-투-텍스트로

T5는 인코더-디코더 구조로 번역, 요약, 분류, 질의응답을 모두 텍스트 입력 → 텍스트 출력 형식으로 통일했습니다. 이는 트랜스포머가 범용 문제 해결 프레임이 될 수 있음을 보여줬습니다.

출처: T5 (JMLR), Google Research Blog

ViT — 언어를 넘어 비전으로

ViT는 이미지를 작은 패치로 나눈 뒤 이를 단어 시퀀스처럼 처리했습니다. 즉 Transformer가 문장 전용 구조가 아니라, 시퀀스로 표현 가능한 데이터 전반에 적용 가능한 일반 아키텍처임을 보여줬습니다.

출처: ViT (Google Research), ViT (arXiv)

10. 결론

Transformer의 진짜 의미는, 시퀀스를 시간축의 상태 전달 문제로 보던 관점을 전역 관계 계산 문제로 바꾼 데 있습니다.

그래서 이 구조는 단순 번역 모델을 넘어, BERT의 이해, GPT의 생성, T5의 통합, ViT의 확장으로 이어지며 현대 AI의 공통 기반이 됐습니다.

주요 참고 문서: The Illustrated Transformer, Attention Is All You Need
